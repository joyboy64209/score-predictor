# Match Predictor

AI-powered **Football Match Prediction** platform. Select a league, season and
matchday, then browse upcoming fixtures with high-confidence betting picks
generated by a machine-learning pipeline.

## Architecture

| Layer     | Stack                                                              |
| --------- | ----------------------------------------------------------------- |
| Frontend  | React + TypeScript + Vite + Tailwind + React Router + TanStack Query |
| Backend   | NestJS + Prisma + PostgreSQL + JWT + RBAC + Swagger               |
| ML        | FastAPI + scikit-learn + pandas + joblib                          |
| Ops       | Docker Compose (Postgres, backend, ml, frontend)                  |

```
match-predictor/
 backend/      NestJS API + Prisma schema + seed
 frontend/     React SPA
 ml/           FastAPI ML service (pipeline, engine, providers)
 database/     (Prisma lives in backend/prisma)
 docker/       Dockerfiles + docker-compose.yml
 docs/         Documentation
```

## Features

- League → Season → Matchday selector flow.
- Only fixtures with **at least one qualifying prediction** are shown.
- Accordion fixture cards revealing qualifying picks grouped into:
  - Match Result (1X2)
  - Double Chance (1X / X2 / 12)
  - Combination Bets (17 supported combinations)
  - Other Markets (BTTS, Over/Under, team goals, exact score, …)
- **Configurable confidence thresholds** (Match Result ≥70%, Double Chance ≥90%,
  Other ≥80%, Combination ≥80%) editable from the admin panel.
- Ranked predictions by confidence + expected value, with "Why?" analysis.
- JWT auth + role-based admin panel (sync, retrain, thresholds, metrics, users).
- Provider abstraction so data can come from any football API or CSV.

## Quick start (local, without Docker)

### 1. Database
```bash
cp .env.example .env
# start postgres (or use docker compose postgres only)
cd backend
npm install
npx prisma generate
npx prisma migrate dev --name init
npm run prisma:seed
```

### 2. Backend
```bash
cd backend
npm run start:dev      # http://localhost:3001  (Swagger: /api/docs)
```

### 3. ML service
```bash
cd ml
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000
# train a model first:  POST /train
```

### 4. Frontend
```bash
cd frontend
npm install
npm run dev            # http://localhost:3000
```

Default users (from seed): `admin@predictor.io / admin123` (ADMIN),
`user@predictor.io / user1234` (USER).

## Docker

```bash
cp .env.example .env
docker compose -f docker/docker-compose.yml up --build
```

## Desktop app

```bash
npm install
npm run build:frontend
npm run build:desktop
```

The desktop shell proxies `/api` to `MATCH_PREDICTOR_BACKEND_URL` and defaults to `http://localhost:3001`.

## API

| Method | Path                          | Description                          |
| ------ | ----------------------------- | ------------------------------------ |
| POST   | `/api/auth/login`             | Obtain JWT                           |
| GET    | `/api/leagues`                | List leagues + seasons               |
| GET    | `/api/fixtures`               | Upcoming fixtures (filter params)    |
| GET    | `/api/predictions/fixture/:id`| Ranked, sectioned predictions        |
| GET    | `/api/predictions/thresholds` | Current thresholds                   |
| PUT    | `/api/admin/thresholds`       | Update thresholds (ADMIN)            |
| POST   | `/api/admin/retrain`          | Trigger ML retrain (ADMIN)           |

ML service (`:8000`): `POST /predict`, `POST /train`, `GET /health`.
